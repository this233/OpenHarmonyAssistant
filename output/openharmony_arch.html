OpenHarmony 技术架构

分层设计，模块化构建，赋能万物互联

架构总览

内核层 (Kernel Layer)

  • 内核子系统

    采用多内核设计，支持 Linux 内核或 LiteOS，提供基础的进程管理、内存管理等核心能力。

  • 驱动子系统

    基于 HDF (Hardware Driver Foundation) 驱动框架，实现硬件生态的开放与统一。

框架层 (Framework Layer)

  • Ability 框架

    为应用开发提供组件模型，支持 Page、Service、Data 等多种 Ability 类型。

  • ArkUI 框架

    面向 JS/TS 开发者的声明式 UI 框架，支持构建高性能、高流畅的用户界面。

  • 多语言 API

    提供 C/C++/JS 等多语言的系统服务接口，方便开发者调用底层能力。

代码

@Component
struct Index {
  @State message: string = 'Hello World'

  build() {
    Column() {
      Text(this.message)
        .fontSize(24)
        .onClick(() => {
          this.message = 'Hello OpenHarmony!'
        })
    }
    .width('100%')
    .height('100%')
  }
}

系统服务层 (System Service Layer)

分布式软总线 分布式数据管理 分布式任务调度 统一用户认证

系统服务层是 OpenHarmony 的核心能力集合，为上层应用提供分布式、安全、一致的系统服务。
