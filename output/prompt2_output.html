OpenHarmony 四层架构

OpenHarmony 采用分层架构设计，从下到上依次为内核层、系统服务层、框架层和应用层。 这种设计支持组件化裁剪，适应从128KiB到xGiB内存资源的设备部署。

内核子系统目录结构

展示OpenHarmony内核子系统的完整目录结构，体现多内核架构和模块化设计理念

kernel/
├── linux
│   ├── common_modules                     # linux内核通用模块仓
│   ├── linux-4.19                         # OpenHarmony linux-4.19 Common kernel
│   ├── linux-5.10                         # OpenHarmony linux-5.10 Common kernel
│   ├── linux-6.6                          # OpenHarmony linux-6.6  Common kernel
│   ├── build
│   │   ├── BUILD.gn                       # 编译框架GN文件
│   │   ├── kernel.mk                      # 内核编译文件
│   │   └── ohos.build                     # 内核编译组件文件
│   └── ...
└── liteos_a               # liteos内核基线代码
    ├── apps               # 用户态的init和shell应用程序
    ├── arch               # 体系架构的目录，如arm等
    ├── drivers            # 内核驱动
    ├── fs                 # 文件系统模块
    ├── kernel             # 进程、内存、IPC等模块
    └── ...
该目录结构展示了Linux和LiteOS-A内核的组织方式，支持多版本内核并存
